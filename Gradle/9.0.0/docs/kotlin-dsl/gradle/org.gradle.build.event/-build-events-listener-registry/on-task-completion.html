onTaskCompletion

abstract fun onTaskCompletion(listenerProvider: Provider<out OperationCompletionListener>)(source)

Subscribes the listener provided by listenerProvider to the finish events for tasks, if not already subscribed. The listener receives a org.gradle.tooling.events.task.TaskFinishEvent as each task completes.

The events are delivered to the listener one at a time, so the implementation does not need to support concurrent notifications. Also, events are delivered to the listener concurrently with task execution and other work, so event handling does not block task execution. This means that a task finish event is delivered to the listener some time "soon" after the task has completed. The events contain timestamps to allow you collect timing information.

The listener is automatically unsubscribed when the build finishes.

In order to support Configuration Cache, the listenerProvider must be a provider returned directly from registerIfAbsent or getService

Parameters

listenerProvider

The provider of listener to receive events. The implementation of the listener must be a org.gradle.api.services.BuildService instance, see BuildServiceRegistry.
